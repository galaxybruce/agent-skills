# 项目分析与文档生成器 (Project Analyzer & Doc Generator)

当用户请求分析当前项目（如 Android 项目）或生成项目文档时，你需要按照以下标准流程执行。该技能致力于生成准确、精简且结构化的多文件 Markdown 文档集。

## 执行流程

### 第一步：信息收集与预分析
1. **获取配置信息**：读取核心构建与配置文件（如 `build.gradle`, `settings.gradle`, `libs.versions.toml`, `AndroidManifest.xml` 或 `package.json` 等，视项目类型而定）。
2. **分析目录结构**：使用文件浏览工具或命令行（如 `find`）浏览核心业务模块，识别主要的包结构和分层。
3. **定位核心文件**：阅读项目的入口文件（如 `Application` 类、主 Activity/Main 文件）、核心路由、依赖注入配置类以及网络请求定义接口。
4. **排除无关目录**：在分析时主动忽略 `.venv`, `dist`, `node_modules`, `.git`, `build`, `.gradle` 等无关目录。

### 第二步：生成文档集
在项目根目录创建或更新 `docs/` 文件夹，并生成以下 5 个纯文本（Markdown）格式的文档（**严禁使用 emoji 和装饰性符号**）：

1. **PROJECT_OVERVIEW.md (项目概述与文档索引)**
   - 列出项目的主要特性。
   - 简要介绍项目的核心功能、业务领域。
   - 作为 `docs/` 目录的入口和索引文件。
   - 包含其他 4 个文档的链接和简要说明，将分文件组织起来。
   
2. **TECH_STACK.md (技术栈)**
   - 按特征分类（如：UI 框架、架构与数据流、持久化与网络、三方服务集成、常用工具库等）详细列出项目使用的技术栈。

3. **ARCHITECTURE.md (项目架构)**
   - 说明项目的整体架构设计（如组件化、模块化层级）。
   - 阐述模块间关系与数据流向（如 MVVM/UDF 单向数据流）。
   - **必须**使用 mermaid 语法生成架构图。
   - **必须**增加关键组件介绍章节，简要说明核心框架（如 UI 框架、网络库、依赖注入、本地存储等）及其在架构中的作用。

4. **DIRECTORY_STRUCTURE.md (目录结构)**
   - 介绍项目的目录模块，内容分为 3 部分：
     1. **目录整体结构介绍**：以目录树代码块形式展示项目的顶层模块与核心结构，并**必须包含根目录下所有文件的介绍**。
     2. **逐个详细介绍每个模块下的目录**：以目录树代码块形式逐层展示每个目录。
     3. **关键目录说明**：突出说明对业务逻辑、架构支撑起关键作用的特定目录。
   - 突出核心业务逻辑文件的分布。

5. **CORE_FILES.md (核心文件说明)**
   - 重点介绍并提供文件链接（Markdown Link）：项目入口文件、配置文件、核心业务逻辑实现、数据模型与 API 接口、关键组件和服务模块。

### 第三步：更新根目录引用
1. 检查项目根目录的主 `README.md`。
2. 在根目录 `README.md` 中添加或更新**项目文档导航**章节，提供指向 `docs/PROJECT_OVERVIEW.md` 的直接链接，确保分文件在主文件中有引用。

## 严格规则与要求

- **绝对真实**：必须实际调用 `Read` 工具阅读文件内容来了解功能，**严禁基于文件名猜测**。对于现有文件必须重新阅读分析以防内容已更新。
- **校验更新**：对之前生成的文档内容进行全面校验，识别并更新过时信息。
- **验证链接**：确保生成的文档中所有文件路径、目录结构的准确性，避免遗漏或错误的引用。
- **内容风格**：内容要既准确又简洁，避免冗余描述。面向不同技术水平的读者提供必要的背景说明，重点突出项目的独特设计和技术亮点。
- **格式规范**：使用逻辑清晰的层次结构，保持各部分的关联性和完整性。
